## 方案一：使用 Vercel 或 Railway（最简单）

### 使用 Railway（推荐）

Railway 提供免费额度，非常适合部署 Node.js 应用。

1. **准备代码**
   - 将项目推送到 GitHub 仓库
   - 确保项目根目录有 `package.json` 和 `server.js`

2. **注册 Railway**
   - 访问 https://railway.app
   - 使用 GitHub 账号登录

3. **部署项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择你的微信聊天项目仓库
   - 点击 "Deploy Now"

4. **配置环境变量（可选）**
   - 在 Railway 项目设置中添加环境变量
   - `PORT` 会自动配置

5. **获取公网域名**
   - 部署完成后，Railway 会自动分配一个域名
   - 也可以绑定自定义域名

### 使用 Vercel

Vercel 主要用于前端，但也可以部署 Node.js API。

1. 准备代码并推送到 GitHub
2. 访问 https://vercel.com 并登录
3. 导入项目并部署

## 方案二：使用 Docker + 云服务器

### 前置要求

- 一台云服务器（阿里云、腾讯云、AWS 等）
- 服务器已安装 Docker 和 Docker Compose

### 部署步骤

1. **连接到服务器**
   ```bash
   ssh root@your-server-ip
   ```

2. **克隆或上传项目**
   ```bash
   git clone your-repo-url
   cd openclaw
   ```

3. **使用 Docker Compose 启动**
   ```bash
   docker-compose up -d
   ```

4. **查看日志**
   ```bash
   docker-compose logs -f
   ```

5. **配置防火墙**
   - 确保云服务器安全组开放 3000 端口

6. **配置 Nginx 反向代理（可选，推荐）**

   创建 Nginx 配置文件 `/etc/nginx/sites-available/wechat-chat`：
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;

       location / {
           proxy_pass http://localhost:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

   启用配置：
   ```bash
   sudo ln -s /etc/nginx/sites-available/wechat-chat /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl reload nginx
   ```

7. **配置 SSL（使用 Let's Encrypt）**
   ```bash
   sudo apt install certbot python3-certbot-nginx
   sudo certbot --nginx -d your-domain.com
   ```

## 方案三：直接部署到云服务器

### 前置要求

- 一台云服务器
- 已安装 Node.js 16+ 和 npm

### 部署步骤

1. **连接到服务器**
   ```bash
   ssh root@your-server-ip
   ```

2. **安装 Node.js（如果未安装）**
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs
   ```

3. **克隆或上传项目**
   ```bash
   git clone your-repo-url
   cd openclaw
   ```

4. **安装依赖**
   ```bash
   npm ci --only=production
   ```

5. **使用 PM2 管理进程**

   安装 PM2：
   ```bash
   npm install -g pm2
   ```

   启动应用：
   ```bash
   pm2 start server.js --name wechat-chat
   ```

   设置开机自启：
   ```bash
   pm2 startup
   pm2 save
   ```

   查看状态：
   ```bash
   pm2 status
   pm2 logs wechat-chat
   ```

6. **配置防火墙和 Nginx**
   - 同方案二的步骤 5-7

## 方案四：使用 ngrok 内网穿透（临时方案）

适合本地测试和演示，不推荐生产环境使用。

### 使用步骤

1. **下载并安装 ngrok**
   - 访问 https://ngrok.com 下载
   - 或使用包管理器安装

2. **启动本地服务器**
   ```bash
   npm start
   ```

3. **启动 ngrok**
   ```bash
   ngrok http 3000
   ```

4. **获取公网地址**
   - ngrok 会显示一个公网 URL，例如：`https://abc123.ngrok.io`
   - 任何人都可以通过这个 URL 访问你的应用

## 重要注意事项

### 1. 安全性

- **修改 JWT 密钥**：在生产环境中，务必修改 `server.js` 中的 `JWT_SECRET` 为强密钥
- **使用 HTTPS**：生产环境务必使用 HTTPS
- **定期更新依赖**：保持 npm 包更新以修复安全漏洞

### 2. 数据持久化

当前项目使用内存数据库，重启后数据会丢失。生产环境建议：

- 使用 Redis 或 MongoDB 替代内存存储
- 定期备份上传的文件

### 3. 性能优化

- 启用 gzip 压缩
- 配置 CDN 加速静态资源
- 使用 PM2 集群模式

## 常见问题

### Q: 部署后 Socket.IO 连接失败？

A: 确保 Nginx 配置正确支持 WebSocket 升级，参考方案二中的 Nginx 配置。

### Q: 上传的文件重启后丢失？

A: 当前使用内存存储，需要配置持久化数据库，或使用 Docker volumes 持久化 uploads 目录。

### Q: 如何绑定自定义域名？

A: 在云服务商 DNS 管理中添加 A 记录，指向服务器 IP，然后配置 Nginx。
